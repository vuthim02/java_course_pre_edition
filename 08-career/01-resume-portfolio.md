## Resume Rules

1. **One page** (unless 15+ years experience)
2. **Quantify everything** — "Reduced latency by 60%", not "Improved performance"
3. **Use action verbs** — Led, Built, Designed, Implemented, Optimized
4. **Tailor to the job** — highlight relevant skills
5. **Spell check** — ONE typo can disqualify you

### README Best Practices

Every project repo should have:
- **Clear name and description** — what does it do?
- **Tech stack badges** — Java 21, Spring Boot 3.2, etc.
- **Architecture diagram** — ASCII or image
- **Setup instructions** — exactly how to run
- **Tests** — show coverage, pass/fail
- **Link to live demo** — deployed app URL

### Open Source Contribution Strategy

1. **Start small** — fix typos, add tests, improve docs
2. **Build up** — fix bugs, add minor features
3. **Go deep** — contribute to modules you use daily
4. **Be consistent** — 1 contribution/week is better than 10 in a day

Good repos to contribute to:
- `spring-projects/spring-boot`
- `spring-projects/spring-security`
- `testcontainers/testcontainers-java`
- `iluwatar/java-design-patterns`
- `eugenp/tutorials`
