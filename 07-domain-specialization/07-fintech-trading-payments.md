## Payment Processing

### Idempotency (Critical!)

```java
@Service
public class PaymentService {

    @Autowired
    private PaymentRepository paymentRepository;
    @Autowired
    private PaymentGateway gateway;

    public PaymentResult processPayment(PaymentRequest request) {
        // Idempotency key — retry safely without double-charging
        String idempotencyKey = request.idempotencyKey();

        // Check if already processed
        Optional<Payment> existing = paymentRepository.findByIdempotencyKey(idempotencyKey);
        if (existing.isPresent()) {
            return PaymentResult.from(existing.get());  // Return existing result
        }

        // Process payment
        Payment payment = new Payment(request);
        paymentRepository.save(payment);  // Status: PENDING

        try {
            GatewayResponse response = gateway.charge(request);
            payment.complete(response.transactionId());
            paymentRepository.save(payment);
            return PaymentResult.success(payment);
        } catch (Exception e) {
            payment.fail(e.getMessage());
            paymentRepository.save(payment);
            return PaymentResult.failure(e.getMessage());
        }
    }
}

// Request with idempotency
public record PaymentRequest(
    String idempotencyKey,   // UUID generated by client
    String userId,
    Money amount,
    PaymentMethod method
) {}
```

### Double-Entry Ledger

Every financial transaction MUST debit one account and credit another.

```java
@Service
@Transactional
public class LedgerService {

    @Autowired
    private AccountRepository accountRepository;
    @Autowired
    private TransactionRepository transactionRepository;

    public void transfer(AccountId from, AccountId to, Money amount) {
        // 1. Lock accounts (prevent race conditions)
        Account debitAccount = accountRepository.findByIdWithLock(from);
        Account creditAccount = accountRepository.findByIdWithLock(to);

        // 2. Validate sufficient funds
        if (debitAccount.getBalance().compareTo(amount) < 0) {
            throw new InsufficientFundsException();
        }

        // 3. Create double-entry transaction
        Transaction transaction = new Transaction();
        transaction.addEntry(new LedgerEntry(
            debitAccount.getId(), amount.negate(), EntryType.DEBIT
        ));
        transaction.addEntry(new LedgerEntry(
            creditAccount.getId(), amount, EntryType.CREDIT
        ));

        // 4. Update balances
        debitAccount.debit(amount);
        creditAccount.credit(amount);

        // 5. Save (atomic — all or nothing)
        accountRepository.save(debitAccount);
        accountRepository.save(creditAccount);
        transactionRepository.save(transaction);

        // 6. Publish event
        eventPublisher.publish(new TransferCompletedEvent(transaction));
    }
}

// Ledger entry — immutable, auditable
@Entity
public class LedgerEntry {
    @Id private UUID id;
    private UUID accountId;
    private BigDecimal amount;  // Positive = credit, Negative = debit
    private EntryType type;
    private LocalDateTime createdAt;
    private UUID transactionId;  // Links debit + credit entries
}
```

## Trading Engine

### Order Book

```java
public class OrderBook {

    private final TreeMap<BigDecimal, List<Order>> buyOrders = new TreeMap<>(Comparator.reverseOrder());
    private final TreeMap<BigDecimal, List<Order>> sellOrders = new TreeMap<>();

    public synchronized List<Trade> placeOrder(Order order) {
        List<Trade> trades = new ArrayList<>();

        if (order.side() == OrderSide.BUY) {
            // Match against sell orders (lowest price first)
            var iterator = sellOrders.entrySet().iterator();
            while (iterator.hasNext()) {
                var entry = iterator.next();
                if (entry.getKey().compareTo(order.price()) > 0) break;

                List<Order> matchingOrders = entry.getValue();
                trades.addAll(match(order, matchingOrders));

                if (matchingOrders.isEmpty()) iterator.remove();
                if (order.isFilled()) break;
            }

            // If not filled, add to order book
            if (!order.isFilled()) {
                buyOrders.computeIfAbsent(order.price(), k -> new ArrayList<>()).add(order);
            }
        } else {
            // Match against buy orders (highest price first)
            // ... symmetric logic
        }

        return trades;
    }

    private List<Trade> match(Order incoming, List<Order> existing) {
        List<Trade> trades = new ArrayList<>();
        var iterator = existing.iterator();

        while (iterator.hasNext() && !incoming.isFilled()) {
            Order resting = iterator.next();
            long quantity = Math.min(incoming.remainingQuantity(), resting.remainingQuantity());

            trades.add(new Trade(
                incoming.id(), resting.id(),
                resting.price(), quantity, Instant.now()
            ));

            incoming.fill(quantity);
            resting.fill(quantity);

            if (resting.isFilled()) iterator.remove();
        }

        return trades;
    }
}
```

@ChannelHandler.Sharable
public class MarketDataHandler extends SimpleChannelInboundHandler<MarketDataPacket> {

    private final OrderBook orderBook;

    @Override
    protected void channelRead0(ChannelHandlerContext ctx, MarketDataPacket packet) {
        switch (packet.type()) {
            case NEW_ORDER -> orderBook.placeOrder(packet.order());
            case CANCEL -> orderBook.cancelOrder(packet.orderId());
            case MODIFY -> orderBook.modifyOrder(packet.orderId(), packet.newPrice());
        }

        // Broadcast updated market state to subscribers
        broadcastOrderBookSnapshot();
    }

## Reconciliation

Reconciliation ensures your records match the bank/gateway's records.

```java
@Service
public class ReconciliationService {

    public ReconciliationResult reconcile(List<Transaction> ourRecords,
                                           List<GatewayTransaction> gatewayRecords) {
        List<Discrepancy> discrepancies = new ArrayList<>();

        // Find transactions in our system but not in gateway
        for (Transaction ours : ourRecords) {
            boolean matched = gatewayRecords.stream()
                .anyMatch(g -> g.transactionId().equals(ours.getGatewayRef()));
            if (!matched) {
                discrepancies.add(new Discrepancy(
                    ours.getId(), "Missing from gateway"
                ));
            }
        }

        // Find transactions in gateway but not in our system
        for (GatewayTransaction theirs : gatewayRecords) {
            boolean matched = ourRecords.stream()
                .anyMatch(o -> o.getGatewayRef().equals(theirs.transactionId()));
            if (!matched) {
                discrepancies.add(new Discrepancy(
                    theirs.transactionId(), "Missing from our system"
                ));
            }
        }

        // Check amounts match
        for (Transaction ours : ourRecords) {
            gatewayRecords.stream()
                .filter(g -> g.transactionId().equals(ours.getGatewayRef()))
                .findFirst()
                .ifPresent(g -> {
                    if (ours.getAmount().compareTo(g.amount()) != 0) {
                        discrepancies.add(new Discrepancy(
                            ours.getId(),
                            "Amount mismatch: ours=" + ours.getAmount()
                            + " gateway=" + g.amount()
                        ));
                    }
                });
        }

        return new ReconciliationResult(discrepancies);
    }
}
```

## Key FinTech Patterns

| Pattern | Description | Implementation |
|---------|-------------|----------------|
| **Idempotency** | Same request processed once | Idempotency key, check before processing |
| **Double-Entry** | Every transaction debits one account, credits another | Ledger with two linked entries |
| **Immutable Ledger** | Never modify past entries, only add new ones | `INSERT` only, no `UPDATE` on transactions |
| **Saga** | Distributed transaction across services | Choreography/Orchestration with compensating actions |
| **Audit Trail** | Every state change logged | Event sourcing + immutable log |
| **Balance Check** | Validate funds before transactions | Pessimistic/optimistic locking |
| **Reconciliation** | Match records with external systems | Batch comparison process |
